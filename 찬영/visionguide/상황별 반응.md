# 상황별 반응

## 음성 우선순위

> 여러 상황이 동시에 발생하면 아래 순서대로 높은 우선순위 음성이 먼저 발화되고, 낮은 순위는 대기하거나 생략됨

| 순위 | 상황 | 예시 | TTS 모드 | 중복 처리 |
|---|---|---|---|---|
| 1 | 장애물 긴급 (1m 이하) | "위험! 앞 차량 매우 가깝습니다." | FLUSH (즉시 끊고 발화) | 같은 라벨 3초 쿨다운 |
| 2 | 배터리 5% / 카메라 가림 경고 | "배터리가 5퍼센트입니다. 안전한 곳으로 이동하세요." | FLUSH | 5초마다 반복 |
| 3 | 횡단보도 진입 | "횡단보도입니다. 멈추고 신호를 확인하세요." | FLUSH | 1회 |
| 4 | 장애물 5m 진입 | "왼쪽 2.3m 차량 접근" | ADD | 같은 라벨당 1회 |
| 5 | 차도 / 자전거도로 진입 | "차도입니다. 주의하세요." | ADD | frontCls 변화 시만 |
| 6 | 골목 / 주의구역 진입 | "골목길입니다. 주의하세요." | ADD | frontCls 변화 시만 |
| 7 | 인도 진입 | "인도입니다." | ADD | frontCls 변화 시만 |
| 8 | 차도 / 자전거도로 5초 반복 유도 | "차도입니다. 왼쪽 인도로 이동하세요." | ADD (다른 음성 진행 중이면 스킵) | — |
| 9 | 배터리 30% / 15% 알림 | "배터리가 30퍼센트 남았습니다." | ADD | 1회 |

### 진동 정책

> 음성과 별개로 진동도 폭주하지 않도록 코얼레싱

| 등급 | 진폭 | 사용 상황 |
|---|---|---|
| 약 | 100 | 5m 진입 (차량/개인이동장치) |
| 중 | 180 | 3m 이내 |
| 강 | 255 | 1m 이하 긴급 |

- 한 번 진동이 끝나기 전(약 1초) 새 진동은 무시 (단, 강 등급은 약·중을 끊고 즉시 실행)
- 100ms 이내 여러 진동 호출이 동시에 오면 가장 강한 것만 실행

## 장애물 (물체)

| 그룹 | 하위 항목 | 5m 이내 진입 | 3m 이내 | 1m 이하 |
|---|---|---|---|---|
| **차량** | 승용차, 버스, 트럭, 오토바이 | 음성 1회 + 진동(약) | 진동(중) | 긴급 음성 1회 + 진동(최대) |
| **개인이동장치** | 자전거, 스쿠터 | 음성 1회 + 진동(약) | 진동(중) | 긴급 음성 1회 + 진동(최대) |
| **사람/동물** | 사람, 고양이, 개, 유모차, 휠체어, 리어카, 이동식 안내판 | 없음 | 없음 | 긴급 음성 1회 |
| **고정장애물** | 바리케이드, 벤치, 볼라드, 의자, 소화전, 키오스크, 주차미터, 기둥, 화분, 전력 제어기, 정류장, 테이블, 가로수, 신호등 제어기 | 없음 | 없음 | 긴급 음성 1회 |
| **신호등/표지판** | 신호등, 교통표지판 | 없음 | 없음 | 진동(최대) |
| 기타 | — | 없음 | 없음 | 없음 |

### 동시 발생 시 처리

> 한 프레임에 여러 객체가 탐지되어도 음성은 1개만, 나머지는 진동으로 보완

| 상황 | 처리 |
|---|---|
| 같은 그룹 ≥ 3개 | "왼쪽에 사람 여러 명" 처럼 개수 대신 "여러"로 일반화 |
| 다른 그룹 동시 | 위험도 점수 = 면적 × 중심 가까움 × 신뢰도 × 그룹 가중치, 1위만 음성 |
| 1m 이하 ≥ 2개 | 가장 가까운 1개만 긴급 음성, 나머지는 강진동 |
| 5m 진입 ≥ 2개 | "여러 장애물 접근. 주의하세요." 1회 |

- 그룹 가중치: 차량(1.0) > 개인이동장치(0.9) > 사람(0.7) > 고정장애물(0.5) > 신호등(0.3)
- 5m 진입 음성은 최대 1.5초, 1m 긴급 음성은 최대 2초 분량 (다음 음성과 충돌 줄이기)

### 방향 유도 음성 (차도 · 자전거도로 5초 반복)

> 카메라로 감지된 왼쪽/오른쪽 노면 정보를 기반으로 동적 생성

| 왼쪽 | 오른쪽 | 유도 음성 |
|---|---|---|
| 인도 or 골목 | 없음 | "{상황} 왼쪽 인도(골목)로 이동하세요." |
| 없음 | 인도 or 골목 | "{상황} 오른쪽 인도(골목)로 이동하세요." |
| 인도 or 골목 | 인도 or 골목 | "{상황} 왼쪽 또는 오른쪽 인도로 이동하세요." |
| 횡단보도 | 없음 or 횡단보도 | "{상황} 왼쪽(오른쪽) 횡단보도 방향으로 이동하세요." |
| 주의구역 | 없음 or 주의구역 | "{상황} 왼쪽(오른쪽) 골목 방향으로 이동하세요." |
| 없음 | 없음 | "{상황} 천천히 멈추고 주변을 확인하세요." |

- {상황} = "차도입니다." 또는 "자전거 도로입니다."

### 횡단보도 신호등 상태

> 횡단보도 진입 후 카메라에 신호등이 보이면 색상을 분석해 안내

| 신호등 상태 | 음성 |
|---|---|
| 초록불 | "초록불입니다. 건너세요." |
| 빨간불 | "빨간불입니다. 멈추세요." |
| 신호등 없음 / 인식 불가 | "신호등을 확인할 수 없습니다. 주변을 살펴보세요." → 왼쪽·오른쪽에 차량·개인이동장치 없으면 "건너셔도 됩니다." |

## 목적지 네비게이션

> 목적지 설정 후 GPS 기반으로 경로 안내, 이탈 시 재탐색

| 상황 | 음성 |
|---|---|
| 올바른 방향으로 이동 중 | "X미터 앞에서 좌회전(우회전)하세요." |
| 반대 방향으로 걷는 중 | "반대 방향입니다. 돌아서세요." |
| 경로 크게 이탈 | "경로를 벗어났습니다. 경로를 재탐색합니다." |
| 경로 재탐색 완료 | "새로운 경로를 안내합니다." |
| 교차로 접근 | "X미터 앞에서 좌회전(우회전)하세요." |
| 목적지 근처 | "목적지가 X미터 앞입니다." |
| 목적지 도착 | "목적지에 도착했습니다." |

### 먼 거리 목적지 (대중교통 추천)

> 도보 경로가 2km 또는 30분을 초과하면 가까운 정류장으로 안내

| 상황 | 음성 |
|---|---|
| 목적지 2km 초과 | "목적지까지 X킬로미터입니다. 도보로 30분 이상 걸립니다. 가까운 정류장으로 안내할까요?" |
| 사용자 응답 대기 (10초) | (응답 없으면) "도보로 안내합니다." |
| 사용자 "네" | T맵 POI 검색 → 반경 500m 내 가장 가까운 버스정류장/지하철역/택시승강장 1개 선택 → "OO 정류장으로 안내합니다." |
| 사용자 "아니오" | "도보로 안내합니다." |
| 정류장 도착 | "OO 정류장에 도착했습니다. 대중교통 이용 후 다시 목적지를 말씀해주세요." |

## 카메라 상태

> 어두운 환경 / 가림 감지 시 손전등 자동 ON, 앱 종료 시 자동 OFF

| 상황 | 음성 | 동작 |
|---|---|---|
| 어두움 5초 지속 (평균 밝기 < 30/255) | "어두워서 잘 보이지 않습니다. 손전등을 켭니다." | 손전등 자동 ON |
| 손전등 ON 후에도 5초 어두움 | "카메라가 가려져 있거나 너무 어둡습니다. 카메라를 확인하세요." | 10초마다 반복 |
| 5초 연속 탐지·세그 둘 다 빈 결과 (밝기 정상인데 가림) | "카메라가 가려진 것 같습니다." | 10초마다 반복 |
| 다시 밝아짐 | "카메라가 정상 작동합니다." | 손전등 자동 OFF |
| 앱 종료 / 백그라운드 진입 | (음성 없음) | 손전등 강제 OFF |

## 배터리 상태

> 배터리가 부족할 때 단계별 알림 + 자동 절전 모드

| 잔량 | 음성 | 동작 |
|---|---|---|
| 30% | "배터리가 30퍼센트 남았습니다." (1회) | — |
| 15% | "배터리가 15퍼센트 남았습니다. 절전 모드로 전환합니다." | 절전 모드 진입 |
| 5% | "배터리가 5퍼센트입니다. 곧 꺼집니다. 안전한 곳으로 이동하세요." | 5초마다 반복 |
| 충전 시작 | "충전이 시작되었습니다." (1회) | 절전 모드 자동 해제 |

### 절전 모드 동작

> 15% 이하 진입 시 자동, 충전 시 자동 해제

| 항목 | 정상 모드 | 절전 모드 |
|---|---|---|
| /detect 호출 (정지 중, GPS 속도 < 0.5m/s) | 매 프레임 | 1초/회 |
| /detect 호출 (보행 중) | 매 프레임 | 매 프레임 유지 |
| /segment 호출 | 2프레임마다 | 비활성화 |
| 차도 5초 반복 음성 | 5초 간격 | 10초 간격 |
| 화면 ON 유지 | 유지 | 해제 (FLAG_KEEP_SCREEN_ON 끄기) |
| 지도 그리기 | 활성 | 마커만 갱신, 타일 다시 그리기 안 함 |
| 나침반 센서 | SENSOR_DELAY_UI | SENSOR_DELAY_NORMAL |

### 다중 사용자 정책

- 서버는 동시 활성 세션을 최대 **2명**까지만 허용
- 활성 세션 기준: 최근 30초 이내 /detect 또는 /segment 요청한 클라이언트
- 새 클라이언트 접속 시 슬롯 ≥ 2이면 `429 Too Many Clients` 응답
- 클라이언트는 30초마다 슬롯 재시도, 비면 자동 복구

| Depth 꺼진 상태에서 장애물 탐지 | "왼쪽(정면/오른쪽)에 {장애물}이 있습니다." | 5m·1m 단계 구분 없이 탐지 즉시 1회 음성 |
| 서버 부하로 자동 OFF | "서버가 바빠 거리 측정을 일시 중단합니다." | 거리 없이 방향만 안내 |
| 서버 부하 해소로 자동 ON | "거리 측정이 다시 활성화되었습니다." | 정상 거리 기반 안내로 복구 |
| Depth 다시 켜짐 (수동/시스템) | "거리 측정이 활성화되었습니다." | 정상 거리 기반 안내로 복구 |

## 배터리 절감 (예정 작업 — 시간순 적용 계획)

### 현재 배터리 소모 구조

| 소모처 | 현재 상태 | 비중 (대략) |
|---|---|---|
| 화면 ON 유지 (`FLAG_KEEP_SCREEN_ON`) | 항상 켜짐, 풀 밝기 | **30~40%** |
| 카메라 + 프레임 분석 | CameraX 프리뷰 + ImageAnalysis ~10fps | 20~25% |
| 네트워크 (/detect + /segment) | 매 프레임 JPEG 업로드 (~30KB × 5~10/s) | 15~20% |
| GPS (`PRIORITY_HIGH_ACCURACY` 1Hz) | 고정밀 1초마다 | 10~15% |
| 나침반 센서 (`SENSOR_DELAY_UI` ~60Hz) | 지도 회전용 | 5~8% |
| TTS · 진동 · 지도 렌더링 | 이벤트성 | 5~10% |

> 화면 + 카메라 + 네트워크가 70% 이상 → 여기를 우선 절감.

### 해결 방안 (적용 순서)

#### 1️⃣ JPEG 해상도 동적 조정 (15~25% 절감)

- 기본 320×240 (저해상) 전송 → 서버 YOLO 입력은 어차피 640으로 리사이즈하므로 정확도 영향 작음
- 위험 객체 탐지 직후 N프레임만 640×480으로 일시 상승 (정밀도 필요한 순간)
- **효과**: 네트워크 + 서버 GPU 동시 절감

#### 2️⃣ 진동 패턴 단축 (2~5% 절감)

- 일반 알림 진동 길이 0.3~0.5초 (현재 ~1초)
- 긴급(1m 이하)만 1초 유지
- **효과**: 진동 모터 전력 절감 + 음성과 충돌 추가 감소

#### 3️⃣ 움직임 기반 적응형 FPS (20~50% 절감)

- **GPS 속도 기반 분기**:
  - 정지 (< 0.5m/s) → /detect 1초/회
  - 보통 보행 → 0.5초/회
  - 빠른 이동 → 0.2초/회
- **장면 변화 감지** (보조): 직전 프레임과 8×8 다운샘플 차이 < 임계치 → 서버 호출 스킵
- **효과**: 신호대기·정지 시간이 많은 도심 보행에서 큰 절감

#### 4️⃣ GPS 정확도 동적 조정 (10~15% 절감)

- 내비 안내 중 → `PRIORITY_HIGH_ACCURACY` 1Hz 유지
- 일반 모드 (목적지 없을 때) → `PRIORITY_BALANCED_POWER_ACCURACY` 5초/회
- 정지 시 추가로 더 띄엄띄엄
- **효과**: GPS 모뎀 active time 감소

#### 5️⃣-A 화면만 검게 표시 (간단 버전, 25~30% 절감) — **현재 적용**

- 사용자가 토글 버튼 누르면 화면 풀스크린 검정 + `screenBrightness = 0.01f`로 백라이트 거의 OFF
- Activity는 살아있으므로 카메라·탐지·음성·진동 정상 동작
- `FLAG_KEEP_SCREEN_ON`은 유지 (자동 꺼짐으로 인한 카메라 release 방지)
- 다시 누르면 원래 밝기로 복귀
- **효과**: 디스플레이 백라이트 전력 거의 다 절감 (~25~30%)
- **구현**: Activity 단독, Service·알림·권한 추가 불필요

#### 5️⃣-B Foreground Service + 진짜 화면 OFF (확장 버전, 30~40% 절감) — **장기 로드맵**

> 5️⃣-A로 충분하지 않을 때, 후속 작업으로 검토.

- 카메라·탐지·음성·진동 로직을 `ForegroundService`로 이전
- 알림 영역에 "SafeStep 안내 중" 영구 알림 표시 (Foreground Service 필수 요구)
- Activity가 paused/destroy 되어도 Service에서 모든 안내 계속
- 사용자가 전원 버튼 눌러 화면 끄거나 자동 꺼짐 허용해도 안내 정상
- 폰을 주머니/가방에 넣은 시나리오에 가장 적합
- ⚠️ Android 14+ `FOREGROUND_SERVICE_CAMERA` 권한 (manifest + 런타임 모두) 필요
- ⚠️ 카메라/탐지 로직을 Service로 옮기는 큰 리팩토링 (1~2일 작업)
- **효과**: A 대비 추가 5~10%, 총 30~40%

### 5️⃣-A vs 5️⃣-B 비교

| | 5️⃣-A (현재) | 5️⃣-B (확장) |
|---|---|---|
| 절감 효과 | 25~30% | 30~40% |
| 구현 복잡도 | 매우 쉬움 | 복잡 |
| Foreground Service | 불필요 | 필요 |
| 알림 영역 표시 | 없음 | "안내 중" 영구 알림 |
| Android 14 권한 | 불필요 | `FOREGROUND_SERVICE_CAMERA` 필요 |
| 폰 주머니에 넣기 | 가능 (화면 켜진 상태) | 가능 (화면 OFF 상태, 더 자연스러움) |

### 누적 적용 시 예상 효과

| 단계 | 누적 절감 | 배터리 1.5h → |
|---|---|---|
| 1️⃣ + 2️⃣ | 약 17~30% | ~2시간 |
| + 3️⃣ + 4️⃣ | 약 45~70% | 3~4시간 |
| + 5️⃣-A | 약 65~80% | 4.5~5시간 |
| + 5️⃣-B (장기) | 약 70~85% | 5~6시간 |
