# 🦮 시각장애인을 위한 실시간 AI 보행 비서

> **"지도가 알려주지 않는 찰나의 위험과 공간의 의미를 목소리로 들려주는 AI 보조자"**

---

## 📌 1. 프로젝트 개요

### 1-1. 프로젝트명
**시각장애인을 위한 실시간 AI 보행 안내 서비스** *(가칭)*

### 1-2. 프로젝트 한 줄 소개
스마트폰 카메라로 들어온 영상을 AI가 분석하여, 시각장애인 사용자에게 **주변 객체, 위험 요소, 거리 정보**를 **실시간 음성**으로 안내하는 보조 서비스

### 1-3. 개발 기간
**2026년 4월 17일 ~ 5월 18일 (약 1개월)**

### 1-4. 팀 구성
**4명 (교육과정: 객체인식 모델과 LLM을 활용한 지능형 시스템 구현)**

## 🎯 2. 주제 선정 배경

### 2-1. 왜 '시각장애인 서비스'인가?

#### ① 사회적 필요성
국내 시각장애인은 약 **25만 명**(2023년 보건복지부 기준)에 달하지만, 이들의 안전한 보행을 돕는 **실시간 AI 보조 서비스**는 여전히 부족한 실정이다.

특히 **보행 중 발생하는 사고**의 상당수는:
- 갑자기 나타나는 자전거·킥보드
- 머리 높이의 공중 장애물 (간판, 나뭇가지)
- 횡단보도 신호 미인지
- 점자블록 위 장애물

등 **'지도 정보로는 알 수 없는 찰나의 위험'**에서 비롯된다.

#### ② 기존 서비스의 한계
| 기존 서비스 | 한계점 |
|------------|--------|
| 음성 내비게이션 | 정해진 경로 안내만 가능, **즉각적인 위험 감지 불가** |
| Google Lookout | 정적 객체 위주 인식, **한국 보행 환경 특화 X** |
| Be My Eyes | 사람이 직접 도와주는 방식, **24시간 실시간 X** |

→ **AI 기술(Vision + LLM)을 통해 '실시간 + 한국 환경 특화 + 자동화'된 솔루션 제공이 필요**

#### ③ 교육과정과의 연계
본 프로젝트는 교육과정에서 학습한 **객체 인식 모델(YOLO)과 LLM 기술을 종합적으로 활용**할 수 있는 주제이며, 단순한 기술 데모를 넘어 **실질적 사회 가치를 창출**할 수 있다는 점에서 선정하였다.

### 2-2. 프로젝트의 차별점

| 구분 | 기존 서비스 | 본 프로젝트 |
|------|------------|------------|
| 인식 대상 | 정적 객체 위주 | **이동 물체 + 한국형 도로 시설물** |
| 거리 정보 | 미제공 | **단안 카메라 기반 거리 추정 (Depth Anything V2)** |
| 안내 방식 | 단순 객체명 | **LLM 기반 자연스러운 상황 설명 + 행동 제안** |
| 사용 환경 | 실외 위주 | **실외 보행 + 실내 길 안내(RAG)** |

> **핵심 차별점**: *"지도에 없는 찰나의 위험을 AI가 음성으로 번역한다"*

## 📊 3. 사용 데이터 및 모델 선정

### 3-1. 메인 데이터셋: AI Hub '인도 보행 영상'

#### ✅ 왜 이 데이터를 선택했는가?

**① 한국 보행 환경에 최적화된 데이터**
- COCO, ImageNet 등 일반 데이터셋은 **한국의 도로 시설물(점자블록, 볼라드, 가로수 보호대 등)** 을 학습하지 못함
- AI Hub '인도 보행 영상' 데이터셋은 **국내 보도 환경에서 직접 수집**된 데이터로, 시각장애인 보행 안전에 직결되는 객체들이 라벨링되어 있음

**② 시각장애인 보행 안전 관련 객체 포함**
- 점자블록, 볼라드, 가로수, 맨홀
- 신호등, 횡단보도, 정류장
- 자전거, 킥보드, 사람 등 이동 객체

**③ 공공 데이터로서의 신뢰성**
- 정부 공인 데이터 → 학술/사회적 가치 명확
- 라벨링 품질 검증 완료

**④ Surface Masking 데이터셋은 제외한 이유**
- 세그멘테이션 라벨링 작업 부담 과중
- 1개월 프로젝트 기간 내 학습/활용 어려움
- → **객체 검출(Object Detection)** 데이터에 집중하여 효율성 확보

### 3-2. 사용 AI 모델

#### ① 객체 인식: **YOLOv11**
- AI Hub 데이터로 **파인튜닝(Fine-tuning)** 진행
- 경량 모델(YOLOv11-nano) 사용으로 추론 속도 확보
- 시각장애인 보행 환경 특화 객체 인식

#### ② 거리 추정: **Depth Anything V2 (Small)**
- **NeurIPS 2024 채택 모델** → 학술적 신뢰성
- **Apache-2.0 라이선스** → 자유로운 사용 가능
- **24.8M 파라미터의 경량 모델** → 노트북 환경에서도 동작
- 단안 카메라(Monocular)로 깊이(Depth) 추정 → **별도 하드웨어(LiDAR) 불필요**

#### ③ 자연어 생성: **Gemini 2.0 Flash**
- LangChain을 통한 프롬프트 엔지니어링
- 객체 + 거리 + 상황 정보 → **자연스러운 음성 안내문 생성**

#### ④ 음성 변환: **gTTS / Web Speech API**
- 한국어 TTS 지원
- 무료 라이브러리로 구현 비용 절감

#### ⑤ 신호등 색상 인식: **OpenCV (HSV 색공간 분석)**
- YOLO로 신호등 검출 → OpenCV로 빨간불/초록불 구분
- 별도 모델 학습 불필요, 빠른 처리 속도

## 🛠️ 4. 시스템 아키텍처

### 4-1. 전체 구조

```
┌─────────────────────────┐
│   📱 사용자 (스마트폰)    │
│  ┌───────────────────┐  │
│  │  웹브라우저 접속   │  │
│  │                   │  │
│  │  [입력 모드 선택]  │  │
│  │  ① 카메라 실시간   │  │
│  │  ② 영상 파일 분석  │  │
│  └─────────┬─────────┘  │
└────────────┼────────────┘
             │ (이미지/영상 전송)
             ▼
┌─────────────────────────────────┐
│   💻 노트북 서버 (FastAPI)       │
│                                  │
│  ┌──────────────────────────┐  │
│  │  ① YOLO (객체 인식)       │  │
│  │  ② Depth Anything (거리)  │  │
│  │  ③ OpenCV (신호등 색상)   │  │
│  │  ④ RAG (위치 기반 검색)   │  │
│  │  ⑤ Gemini LLM (안내 생성) │  │
│  │  ⑥ gTTS (음성 변환)       │  │
│  └──────────────────────────┘  │
└────────────┬────────────────────┘
             │ (음성 안내 전송)
             ▼
┌─────────────────────────┐
│  📱 스마트폰 음성 출력   │
│      🔊 "1시 방향        │
│      자전거 접근..."     │
└─────────────────────────┘
```

### 4-2. 입력 모드 (2가지)

본 시스템은 **두 가지 입력 모드**를 모두 지원한다.

#### 모드 1: 스마트폰 카메라 (실시간)
- 사용자가 스마트폰 웹브라우저에서 카메라 활성화
- 일정 간격(2~3초)으로 프레임을 서버에 전송
- 서버 분석 후 음성 안내를 스마트폰에서 재생
- **실제 사용 환경** 시연용

#### 모드 2: 영상 파일 분석
- 사전 촬영된 영상을 서버에 업로드
- 프레임 단위로 분석 후 결과 반환
- 스마트폰에서 음성 출력
- **발표 시연 안정성** 확보용

### 4-3. 처리 흐름 (Pipeline)

```
[입력 영상/이미지]
       ↓
🔍 Vision AI Layer
   ├─ YOLO: "사람, 자전거, 점자블록 감지"
   ├─ Depth Anything: "사람 2.1m, 자전거 1.4m"
   └─ OpenCV: "신호등 = 빨간불"
       ↓
🧠 RAG Layer (선택적)
   └─ 위치 기반 정보 검색 (실내 안내 시)
       ↓
🗣️ LLM Layer (Gemini)
   └─ "1시 방향 1.4m에 자전거가 접근 중입니다.
       잠시 멈춰 우측으로 이동하세요."
       ↓
🔊 TTS Layer (gTTS)
   └─ 음성 파일 생성 → 스마트폰 재생
```

## 🎬 5. 핵심 기능 및 시나리오

### 5-1. 기능 우선순위

| 우선순위 | 기능 | 설명 |
|---------|------|------|

### 5-2. 사용 시나리오

#### 📍 시나리오 1: 보행 중 위험 감지
```
[상황] 시각장애인이 길을 걷는 중, 자전거가 빠르게 접근
[시스템 동작]
  → YOLO: 자전거 감지
  → Depth Anything: 1.4m 거리 측정
  → Gemini: 안내문 생성
  → TTS: "⚠️ 1시 방향 1.4m에 자전거 접근 중입니다.
          잠시 멈추고 우측으로 한 걸음 이동하세요."
```

#### 📍 시나리오 2: 횡단보도 신호 인식
```
[상황] 횡단보도 앞에 도착
[시스템 동작]
  → YOLO: 횡단보도 + 신호등 감지
  → OpenCV: 신호등 색상 분석 (빨간불)
  → TTS: "횡단보도입니다. 현재 빨간불, 잠시 기다려주세요."
  
  [신호 변경]
  → OpenCV: 초록불 감지
  → TTS: "초록불로 바뀌었습니다. 직진하세요."
```

#### 📍 시나리오 3: 실내 길 안내 (RAG)
```
[상황] 학원 건물 내부에서 화장실을 찾는 중
[사용자 음성] "화장실 어디야?"
  → RAG: 학원 내부 정보 검색
  → Gemini: "직진 5미터 후 좌측에 화장실이 있습니다.
            벽을 따라 이동하세요."
```

## 🎤 9. 시연 방식

### 9-1. 발표 시연 시나리오

본 프로젝트는 **두 가지 시연 방식**을 활용한다.

#### ① 메인 시연: 영상 파일 분석 모드 (안정성 확보)
- 사전 촬영된 영상을 노트북에서 재생
- 서버가 영상 프레임을 분석
- 분석 결과는 **노트북 화면에 시각적으로 표시**
- 동시에 **스마트폰에서 음성 안내 재생**

> 💡 **발표 효과**: *"노트북 화면은 청중이 보고, 스마트폰 음성은 시각장애인 사용자가 듣는 것"* → 시각장애인의 입장을 직관적으로 전달

#### ② 보조 시연: 스마트폰 카메라 라이브 모드
- 발표장 안에서 카메라로 직접 시연
- "실시간 작동" 신뢰성 입증
